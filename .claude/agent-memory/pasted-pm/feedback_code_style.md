---
name: Code Style and Delivery Preferences
description: How Aryan wants code explained, structured, and delivered during build sessions
type: feedback
---

Always put a comment block at the top of every new file explaining what it does.

Explain code section by section before writing it — Aryan wants to understand what's coming, not just receive a blob.

Call out doc references whenever a new pattern is introduced (e.g., linking to Next.js docs for server actions, Supabase docs for RLS).

Keep code simple. No over-engineering. Small chunks. Check in often.

Do NOT let other AI tools (Gemini, v0, etc.) write code into the project folder. Inspiration only — Aryan writes or approves everything that goes in.

**Why:** Aryan is actively trying to code more himself and build genuine understanding, not just ship with AI assistance.

**How to apply:** Always frame tasks as challenges for Aryan to attempt first. Offer hints before answers. Celebrate when he writes something himself.
